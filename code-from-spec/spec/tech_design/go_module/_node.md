# ROOT/tech_design/go_module

## Intent

Documents the Go module configuration (`go.mod`) and how
it is managed.

## Contracts

The module path is:
`github.com/CodeFromSpec/tool-staleness-check`

## Constraints

- `go.mod` is not generated by a spec node. It is managed
  by Go tooling.
- After code generation, run `go mod tidy` to update
  dependencies based on the imports in generated code.
- `go.sum` is generated automatically by Go tooling —
  never edit it manually.

## Context

For a new project, the module must be initialized before
any code generation:

```
go mod init github.com/CodeFromSpec/tool-staleness-check
```

This is a one-time setup step. After initialization,
`go mod tidy` maintains the module file.
